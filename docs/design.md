# Design Docs

This folder contains the design documentation for the **aware** feed reader app. The design work is organized into focused documents for easier navigation and maintenance.

## Primary Design Documents
- **[Design overview](design-overview.md)** – Vision, core features, and the overall product direction.
- **[Architecture & components](design-architecture.md)** – High-level system architecture, major subsystems, and client/server responsibilities.
- **[Data model & API](design-data-and-api.md)** – Core entities and REST API surface.
- **[UX / UI considerations](ux-design.md)** – UX patterns, navigation, and accessibility guidance.
- **[UI spec (modern/minimalist)](ui-spec.md)** – Visual design tokens, component specs, and screen layouts.
- **[Technical & operational notes](design-technical.md)** – Background processing, scaling, security, extensions, and next steps.

---

> Tip: Start with **Design overview** and then dive into the area you’re working on (UI, API, architecture, etc.).
